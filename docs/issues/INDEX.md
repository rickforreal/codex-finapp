# Defect Backlog Index

This index is the canonical defect backlog for docs-first bug tracking.

## Policy

- One folder per bug: `docs/issues/BUG-####-<slug>/`
- Required files per bug: `ISSUE.md`, `PLAN.md`, `ACCEPTANCE.md`
- Optional file per bug: `NOTES.md`
- Root execution tracking remains in `TASKS.md` and `PROGRESS.txt`.

## Status Vocabulary

- `Open`
- `In Progress`
- `Blocked`
- `Ready for QA`
- `Done`
- `Deferred`

## Severity Vocabulary

- `S0` — production-blocking / data corruption
- `S1` — major feature broken
- `S2` — functional defect with workaround
- `S3` — minor defect / UX polish
